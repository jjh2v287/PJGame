# 개발 로드맵 문서

작성일: 2026-03-14

## 1. 문서 목적

이 문서는 이 프로젝트를 어떤 순서로 구현할지 정리한 실행용 로드맵이다.

다른 문서와의 역할은 아래와 같다.

- `Game_Overview.md`: 이 게임이 무엇인지 정리하는 문서
- `System_Framework.md`: 시스템별 목적과 의존성을 정리하는 문서
- `Weekly_Dev_Log.md`: 매주 실제 작업과 결과를 기록하는 문서
- `Development_Roadmap.md`: 무엇을 어떤 순서로 만들지 정리하는 문서

이 문서를 다시 읽었을 때 아래를 바로 판단할 수 있어야 한다.

- 지금 프로젝트가 어느 단계에 있는가?
- 다음으로 무엇을 만드는가?
- 아직 만들면 안 되는 것은 무엇인가?
- 어떤 기준을 만족하면 다음 단계로 넘어가는가?

## 2. 로드맵 작성 원칙

- 기능 수보다 `플레이 가능한 루프`를 먼저 만든다.
- 코어 루프를 검증하기 전에는 확장 시스템을 붙이지 않는다.
- 각 단계는 가능한 한 `작은 플레이 경험`으로 끝나야 한다.
- 단계마다 `완료 기준`을 둔다.
- 완료 기준을 넘기기 전에는 다음 단계로 무리하게 넘어가지 않는다.

## 3. 전체 개발 단계 요약

현재 기준의 권장 개발 단계는 아래와 같다.

1. 기본 플레이 성립
2. 카라반 원정 루프 성립
3. 자원과 귀환 루프 성립
4. 몬스터 포획 도입
5. 카라반 빌드와 몬스터 활용
6. 경제와 거점 확장
7. 몬스터 성장 심화
8. 엔드 콘텐츠 확장

핵심 원칙은 이렇다.

`먼저 카라반을 지키며 싸우고 돌아오는 재미를 만든 뒤, 그 위에 포획과 성장과 경제를 얹는다.`

## 4. 단계별 상세 로드맵

---

## Phase 1. 기본 플레이 성립

### 목표

플레이어가 움직이고 싸우는 기본 감각을 만든다.

### 구현 항목

- 플레이어 이동
- 기본 공격
- 피격과 체력
- 몬스터 1종
- 기본 전투 한 사이클

### 완료 기준

- 플레이어가 맵에서 자유롭게 움직일 수 있다.
- 몬스터와 싸워 처치할 수 있다.
- 조작감이 기본적으로 불편하지 않다.

### 아직 안 할 것

- 포획
- 교배
- 거래
- 복잡한 자원 경제

---

## Phase 2. 카라반 원정 루프 성립

### 목표

카라반을 지키는 전투가 이 게임의 중심이라는 점을 플레이로 증명한다.

### 구현 항목

- 카라반 추종
- 카라반 체력
- 카라반 피격
- 카라반 파손
- 몬스터가 카라반을 우선 공격하는 규칙
- 카라반 파손 시 손실 처리

### 완료 기준

- 플레이어가 카라반을 의식하며 싸우게 된다.
- 카라반이 맞기 시작하면 실제로 긴장감이 생긴다.
- 카라반 파손이 플레이에 의미 있는 손실을 준다.

### 아직 안 할 것

- 카라반 고급 모듈
- 복잡한 AI 역할군
- 자동 거래 시스템

---

## Phase 3. 자원과 귀환 루프 성립

### 목표

원정에서 얻은 자원을 귀환, 수리, 재도전 동기로 연결한다.

### 구현 항목

- 자원 2~3종
- 채굴/채집 오브젝트
- 루팅/적재
- 거점 또는 안전지대
- 카라반 수리
- 간단한 판매 또는 자원 소모 구조

### 완료 기준

- 플레이어가 자원을 모아 돌아올 이유가 생긴다.
- 모은 자원을 수리에 쓰는 선택이 생긴다.
- 5~10분짜리 `나감 -> 모음 -> 지킴 -> 돌아옴` 루프가 완주된다.

### 아직 안 할 것

- 거점별 가격 차이
- 복잡한 제작 트리
- 이동 상인

---

## Phase 4. 몬스터 포획 도입

### 목표

이 게임의 차별화 요소인 `죽일지 살릴지` 판단을 추가한다.

### 구현 항목

- 몬스터 약화 상태
- 포획 가능 조건
- 포획 장비 1종
- 카라반 보관 슬롯 최소 버전
- 포획 개체 저장

### 완료 기준

- 몬스터를 처치 대신 생포할 수 있다.
- 생포가 전투 중 다른 판단을 요구한다.
- 포획한 개체를 귀환까지 지키는 동기가 생긴다.

### 아직 안 할 것

- 복잡한 포획 도구 종류
- 보스 포획
- 교배/합성

---

## Phase 5. 카라반 빌드와 몬스터 활용

### 목표

포획 몬스터와 카라반이 빌드 선택의 축이 되게 한다.

### 구현 항목

- 카라반 슬롯 2~3종
- 모듈 장착/해제
- 몬스터 1개 전투 보조 또는 패시브 효과
- 적재/방어/포획 중 선택 구조

### 완료 기준

- 서로 다른 카라반 세팅이 플레이 차이를 만든다.
- 몬스터를 보유하는 것이 단순 수집이 아니라 실제 도움이 된다.
- 무엇을 싣고 무엇을 포기할지 선택이 생긴다.

### 아직 안 할 것

- 희귀 모듈 대량 추가
- 다중 몬스터 편성
- 혈통 시스템

---

## Phase 6. 경제와 거점 확장

### 목표

귀환 이후 운영의 폭을 넓혀 장기 플레이 동기를 강화한다.

### 구현 항목

- 거점 2곳 이상
- 거점별 수요 차이
- 기본 교환/판매 구조 확장
- 특정 자원/몬스터의 납품 가치 차등

### 완료 기준

- 같은 자원도 어디에 가져가느냐에 따라 가치가 달라진다.
- 플레이어가 원정 목적지를 시장 기준으로도 고르게 된다.

### 아직 안 할 것

- 이동 상인 복잡화
- 용병 고용
- 복잡한 경제 시뮬레이션

---

## Phase 7. 몬스터 성장 심화

### 목표

장기 수집과 육성의 재미를 만든다.

### 구현 항목

- 몬스터 성장
- 특성 부여
- 같은 계열 교배 또는 단순 합성 1종
- 희귀 결과 개체 생성

### 완료 기준

- 포획한 몬스터를 남길 이유가 생긴다.
- 성장과 조합이 다음 원정의 동기를 만든다.
- 숫자 상승이 아니라 플레이 방식 변화가 느껴진다.

### 아직 안 할 것

- 지나치게 넓은 유전 조합
- 복잡한 혈통 UI
- 대규모 도감 완성

---

## Phase 8. 엔드 콘텐츠 확장

### 목표

후반 플레이의 장기 목표를 만든다.

### 구현 항목

- 희귀 바이옴
- 보스 포획 또는 특수 개체
- 희귀 혈통/돌연변이
- 상위 모듈
- 장거리 상행 또는 특수 계약

### 완료 기준

- 강해진 후에 도전할 콘텐츠가 분명하다.
- 카라반과 몬스터 성장의 최종 목표가 생긴다.

## 5. 현재 기준 즉시 실행 순서

가장 가까운 작업은 아래 순서가 적절하다.

1. 플레이어 이동/공격 안정화
2. 카라반 추종과 카라반 HP 추가
3. 몬스터가 카라반을 노리도록 수정
4. 카라반 파손과 아이템 손실 처리
5. 자원 2~3종 추가
6. 자원 루팅과 카라반 적재
7. 거점 수리
8. 5~10분 플레이 테스트 반복

즉, 당장은 `포획`보다 `카라반을 지키며 자원을 모아 살아 돌아오는 감각`을 먼저 만든다.

## 6. 단계 진입 조건

각 단계는 아래 기준을 만족할 때 다음으로 넘어간다.

### Phase 1 -> Phase 2

- 전투가 플레이 가능한 수준이다.

### Phase 2 -> Phase 3

- 카라반 보호가 실제 플레이 긴장을 만든다.

### Phase 3 -> Phase 4

- 짧은 원정 루프가 완주 가능하다.

### Phase 4 -> Phase 5

- 포획이 처치와 다른 판단을 만든다.

### Phase 5 -> Phase 6

- 카라반 세팅 차이가 플레이 차이로 체감된다.

### Phase 6 -> Phase 7

- 경제와 귀환이 안정적으로 작동한다.

### Phase 7 -> Phase 8

- 몬스터 육성이 장기 목표로 작동한다.

## 7. 보류 우선순위

매력적이지만 늦게 붙여도 되는 요소는 아래와 같다.

- 이동 상인/NPC 거래
- 복잡한 시장 경제
- 용병 시스템
- 고급 교배/합성
- 대규모 혈통 조합
- 다양한 거점 이벤트

이 요소들은 게임을 풍부하게 만들 수 있지만, 초반 코어 재미를 증명하는 데 필수는 아니다.

## 8. 문서 사용법

이 문서는 고정된 정답표가 아니다.

프로젝트가 진행되며 아래 상황이 오면 수정한다.

- 특정 시스템이 예상보다 너무 어렵거나 가치가 낮을 때
- 코어 루프의 재미가 예상과 다를 때
- 일정상 우선순위를 줄여야 할 때

다만 문서를 수정할 때도 아래 원칙은 유지한다.

- 코어 루프 먼저
- 차별화는 그 다음
- 확장은 마지막
